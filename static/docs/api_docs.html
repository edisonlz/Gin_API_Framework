    API使用规则
    1. 点击右侧菜单，选择API分类，如(用户信息 API)。
    2. 点击对应API，如( 登录接口 )，选择API分类。
    3. 展开API分类，其中：POST /live/login 代表， Method URL。
    4. 填写相关参数信息。
    5. 点击Request发起请求，返回数据在数据栏中。
    API接口安全规则
    Method ： GET | POST
    增加参数:
    _s_ ：signature 签名
    _t_:当前时间戳，校队系统时间在初始化接口返回,接口参数有效期30分钟
    timestamp = timestamp + (客户端当前时间 -客户端从初始化获得timestamp的时间)
    这样做到原因是需要客户端校队服务器时间，因为客户端时间不准确。
    签名方式:
    token_string = req_method + ":" + path + ":" + sorted(query) + ":" + timestamp + ":" + secret
    其中：sorted(query) 是按照key的自然顺序排列,然后以key=value的形式累加
    例如: GET /test?b=2&a=1
    token_string = GET + ":" + /test + ":" + a=1b=2 + ":" + 1457665234 + ":" + secret_xxxxxx
    signature = ngx.md5(token_string)
    secret秘钥需要服务端提供，另外客户端需要确保secret安全。
    用户字段描述
    uid ： 显示用户uid, poster:用户海报, logo：头像, cost：花费钻石数量
    _uid：用户id（操作使用），ticket：当前票数，total_ticket:获得过的总票,nickname：昵称，contribute_experience：贡献经验
    desc：用户描述，contribute_ticket：贡献票数，area：地域（目前没有ip库）
    level_desc：等级描述，created_at：创建时间，experience：经验
    follower_count: 关注数，followee_count：粉丝数
    备注 _uid: 统一使用_uid为腾讯云identity，因为uid可以修改(靓号)
    房间字段描述
    name ： 房间名称, member_count:当前房间人数, created_at：创建时间
    is_secret ： 是否私密, love_count:喜欢数量, is_top:是否置顶
    chat_id ： 房间id, key:房间显示号, id:房间id（操作使用）
    max_member_count ：房间最多人数, id:,seq：热度值
    url ：直播url, channel_id:直播频道id
    广场海报图描述
    adv_info ： 广告信息与adv_type一起使用, image:广告图, adv_type：（0 ：'URL',1：'房间id'）
    苹果支付下单流程
    1. 调用下单接口，返回order_id.
    2. 客户端调用支付，此时客户端保存order_id.
    3. 客户端用保存的order_id和支付reciept，到服务器verify验证支付结果
    4. 支付成功进行充值.
    备注：苹果支付下单接口需要传apple_product_id参数
